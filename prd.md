# 📑 Minimal Fit v1.30 상세 설계서

## 1. 제품 개요 (Product Overview)
* **제품명**: Minimal Fit (미니멀 핏)
* **한 줄 정의**: AI 박자 가이드와 자동 동기화 로직을 통해 운동 중 조작을 0에 수렴하게 만드는 지능형 운동 보조 플랫폼.

## 2. 타겟 유저 (Target Audience)
* **Zero-Click User**: 운동 중 무게와 횟수를 일일이 입력하는 것을 번거로워하며 조작을 최소화하고 싶은 사용자.

## 3. 기능 상세 정의 (Feature Specifications)

### [F1] Zero-Click 세트 동기화 (Set Sync)
* **핵심 로직**:
    - 사용자가 **1세트**의 무게나 횟수를 변경하면, **수동 수정(`isEdited`)이 되지 않은 모든 후속 세트**에 해당 값이 즉시 복사됨.
    - 만약 사용자가 특정 세트(예: 2세트)를 직접 수정하면, 해당 세트는 `isEdited: true`가 되어 이후 1세트가 변경되어도 영향을 받지 않음.
* **데이터 구조**: `[{ id: 1, weight: 45, reps: 12, isEdited: false }, ...]`

### [F2] 1초 단위 AI 박자 가이드 (Tempo Engine)
* **핵심 로직**: 운동 중 설정된 박자(1초)에 맞춰 숫자를 소리 내어 카운트함.
* **음성 출력**: Web Speech API(`window.speechSynthesis`)를 활용하여 브라우저 자체 음성으로 "One, Two, Three..." 혹은 "하나, 둘, 셋..." 출력.
* **제어**: 화면 내 'TEMPO' 토글 버튼을 통해 음성 가이드 활성/비활성 가능.

### [F3] 빅 스텝퍼 UI (Interaction)
* **상세**: 수치를 직접 입력하는 대신, 무게와 횟수 입력 칸을 더 크고 직관적인 터치 타겟으로 개선하여 운동 중 조작성 극대화 및 시각적 피드백 강화.

### 기존 기능 리스트
* **[F10~F13] 모바일 최적화 및 세트 로직**: 전체 화면(PWA), 중앙 집중형 레이아웃, 휴식 UI 및 자동 세트 카운팅. [완료]
* **[F8] AI 주도적 무결성 검증**: AI가 업데이트 전 저장소 상태 분석 및 테스트 수행. [활성]
* **[F9] 저장소 청정 유지**: `.gitignore` 적용 완료. [완료]

## 4. 데이터 및 시스템 설계 (System Design)
* **State Management**: `useState`를 통해 세트 데이터 배열(`sets`)을 관리하며, `handleInputChange` 함수 하나로 동기화 로직을 처리함.
* **Layout**: 중앙 프레임 480px 고정 및 모바일 주소창 대응 레이아웃.

## 7. 문서 관리 및 버전 관리 정책
* **버전 체계**: Semantic Versioning 적용 (사용자 승인 시에만 업데이트).
* **PRD 제공 방식**: 변경 시 항상 **본 문서와 같은 전문(Full Text)**을 코드 블록 내에 작성하여 제공함.
* **커밋 메시지 표준**: 모든 업데이트 시 반드시 다음 형식을 준수하며, 제목과 내용 사이에 줄바꿈을 적용함.
    * 형식: `커밋명 [ver] 제목 / 내용 [commit 주요 내용]`
