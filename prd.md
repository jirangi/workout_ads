# 📑 Minimal Fit v1.50 PRD (상세 설계서)

## 1. 제품 개요 (Product Overview)
* **제품명**: Minimal Fit (미니멀 핏)
* **한 줄 정의**: AI 박자 가이드와 자동 동기화 로직을 통해 운동 중 조작을 0에 수렴하게 만드는 지능형 운동 보조 플랫폼.
* **핵심 목표**: 시장 조사부터 출시까지 이어지는 AI 자동화 파이프라인의 첫 번째 표준 모델 확립.

## 2. 타겟 유저 (Target Audience)
* **Heavy User**: 기존 앱의 반복적인 세트 입력에 피로감을 느끼는 중급 이상의 웨이트 트레이너.
* **Returning User**: 저장된 루틴으로 바로 시작하고 싶은 기존 사용자.
* **New User**: 최초 진입 시 숙련도 설정을 통해 맞춤형 루틴을 생성해야 하는 사용자.
* **Tempo Seeker**: 정확한 박자와 템포 유지가 실력 향상에 직결되는 사용자.

## 3. 기능 상세 정의 (Feature Specifications)

### [핵심 엔진 및 로직]
* **[F1] Zero-Click 세트 동기화 (Set Sync)**: 
    - **핵심 로직**: 사용자가 '첫 번째 세트'의 수치를 수정할 경우, `isEdited: false`인 아래의 모든 세트에 데이터 실시간 전파.
    - **데이터 구조**: `sets: Array<{id: number, weight: number, reps: number, isEdited: boolean}>`.
    - **Override 방지**: 이미 개별 수정한 세트는 동기화에서 제외하여 사용자 의도 보호. [v1.15 핵심 복구]
* **[F2] 0.1s 단위 AI 박자 가이드 (Tempo Engine)**: 음성 및 시각적 카운트다운 제공. (현재 실사용성을 위해 1s 단위로 출력 중)
* **[F3] 빅 스텝퍼 UI & 자동 확정 (Interaction)**: 대형 버튼 및 수치 입력창 중앙 배치.

### [수익화 및 리워드]
* **[F16] 일일 광고 리워드 시스템 [v1.50 업데이트]**: 
    - **원칙**: 보상은 휴식 중 광고 노출의 대가로 지급 (수익 배분 플랫폼 70 : 유저 30).
    - **적립 로직**: 휴식(광고) 시청 완료 또는 SKIP 시 30FP 적립.
    - **한도 제약**: 일일 최대 2회까지만 리워드 적립 가능 (부정 수급 방지).
* **[F17] 통합 리워드 샵 (Reward Dashboard)**: 
    - 상단 헤더에 포인트 현황과 샵 진입 버튼을 하나로 통합. [v1.49 반영]
    - 가상 물품 구매 프로세스 시뮬레이션 UI.

### [사용자 경험 및 확장]
* **[F13] 휴식 모드 (Rest UI)**: 60초 타이머와 연동된 다크 모드 전체 화면 오버레이.
* **[F14] 온보딩 및 스마트 홈**: 
    - 최초 진입 시 숙련도 선택(초급/중급/고급/사용자지정) 강제 유도.
    - 홈 화면: [내 루틴 시작하기] 대형 카드 하단에 2분할(50:50) 버튼 배치.
* **[F15] 운동 메타데이터 DB**: `data/exercises.ts`를 통한 종목 대/소분류 관리.

### [인프라 및 자동화]
* **[F6] 배포 파이프라인 안정화**: GitHub Actions 빌드 시 package-lock.json 및 환경 최적화.
* **[F7] 프로젝트 초기화 및 경로 정규화**: 저장소 루트 내 Next.js 필수 구성 파일 배치.
* **[F8] AI 주도적 무결성 검증 (AI Quality Gate)**: AI가 업데이트 전 저장소 상태 분석 및 테스트 수행.
* **[F9] 저장소 청정 유지 (Repository Hygiene)**: .gitignore 적용 완료.

## 4. 데이터 및 시스템 설계 (System Design)
* **프론트엔드**: Next.js 기반 반응형 웹/앱 레이아웃 (중앙 프레임 480px 고정).
* **파일 아키텍처**: layout.tsx(프레임), page.tsx(로직), data/ 폴더 분리 관리.
* **상태 관리**: React `useState`를 활용한 전역 포인트 및 루틴 데이터 관리.
* **배포 정책**: GitHub Pages 배포 (`output: 'export'`, `basePath: '/workout_ads'`).

## 5. AI 자동화 파이프라인 적용 (Automation Pipeline)
* **Step 1 (기획)**: 본 PRD를 통한 기능 명세 자동 갱신.
* **Step 2 (검증/테스트)**: AI가 저장소의 빌드 성공 여부를 상시 모니터링함.
* **Step 3 (개발)**: PRD 명세를 기반으로 AI가 코드 생성.
* **Step 4 (배포)**: GitHub Actions를 통해 자동 반영.

## 6. 수익화 및 확장 계획 (Monetization)
* **광고/구독**: 휴식 시간 전면 광고 및 프리미엄 구독 등급 운영.

## 7. 문서 관리 및 버전 관리 정책
* **버전 체계**: Semantic Versioning 적용.
* **PRD 제공 방식 [필수]**: 변경 시 항상 **전문(Full Text)**을 코드 블록으로 제공.
* **커밋 메시지 표준**: `커밋명 [ver] 제목 / 내용 [commit 주요 내용]` (줄바꿈 준수)
