# 📑 Minimal Fit v1.18 상세 설계서

## 1. 제품 개요 (Product Overview)
* **제품명**: Minimal Fit (미니멀 핏)
* **한 줄 정의**: AI 박자 가이드와 자동 동기화 로직을 통해 운동 중 조작을 0에 수렴하게 만드는 지능형 운동 보조 플랫폼.

## 2. 타겟 유저 (Target Audience)
* **Mobile First**: 운동 중 스마트폰을 바닥이나 거치대에 두고 사용하는 유저.

## 3. 기능 상세 정의 (Feature Specifications)

### [F10] 모바일 전체 화면 및 PWA 설정 [신규]
* **상세**: 브라우저 주소창을 숨기고 홈 화면에 추가 시 앱처럼 작동하도록 `manifest.json` 및 메타 태그를 설정함.
* **효과**: 화면 몰입도 증가 및 운동 중 주소창 오동작 방지.

### [F11] 모바일 중앙 집중형 UI (Centering & Scaling) [신규]
* **상세**: 기존 왼쪽 상단에 쏠려 있던 요소를 화면 정중앙(Vertical & Horizontal Center)에 배치함.
* **가독성**: 텍스트 크기를 대폭 키우고(Big Text), 터치 영역을 확장하여 운동 중에도 쉽게 조작 가능하게 함.

### [F12] 휴식 모드 UI 및 타이머 (Rest Mode) [신규]
* **상세**: '세트 완료' 버튼 클릭 시 즉시 휴식 화면으로 전환되며 설정된 휴식 시간(기본 60초)이 카운트다운됨.
* **전환**: 타이머 종료 시 다시 운동 화면으로 자동 전환 혹은 알림 제공.

### 기존 기능 리스트
* [F1] Zero-Click 세트 동기화 (Set Sync) - [명세 완료]
* [F2] 0.1s 단위 AI 박자 가이드 (Tempo Engine)
* [F3] 빅 스텝퍼 UI & 자동 확정 (Interaction)
* [F6~F9] 인프라 및 무결성 검증 - [완료]

## 4. 데이터 및 시스템 설계 (System Design)
* **Layout**: Next.js `Viewport` 설정을 통해 모바일 주소창 높이 변화에 대응하는 `h-screen` 적용.
* **State**: `isResting` (상태값), `restTimer` (숫자)를 통해 화면 전환 로직 구현.
